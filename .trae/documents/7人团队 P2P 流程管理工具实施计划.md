# 7人团队 P2P 流程管理工具实施计划 (Electron 版)

收到，我们将技术栈切换为 **Node.js + Electron**。这将极大地提升界面的美观度和交互体验，完全能够满足您对“绿色背景”、“红色标识”等视觉反馈的需求，同时保持架构的轻量化。

## 核心架构调整

1.  **技术栈**：
    *   **框架**: Electron (提供跨平台桌面应用能力)
    *   **前端**: HTML5 + CSS3 + JavaScript (原生或轻量级框架，保证极简)
    *   **核心逻辑**: Node.js Native (文件读写、加密)
2.  **优势**：
    *   **UI 自由度高**：可以轻松实现您想要的“每日看板弹窗”、“任务状态颜色标记”等精细交互。
    *   **实时性更强**：利用 Node.js 的 `chokidar` 库监听文件夹，当队友的文件同步过来时，界面可以**毫秒级自动刷新**，无需手动点刷新。

## 实施步骤 (Node.js + Electron)

我将为您构建以下项目结构：

### 阶段 1：项目初始化与基础架构
1.  **`package.json`**: 定义依赖（`electron`, `chokidar` 用于文件监听）。
2.  **`src/main/crypto.js`**: 封装 Node.js 原生 `crypto` 模块（AES-256-GCM），确保高性能和安全性。
3.  **`src/main/file-manager.js`**: 处理加密文件的读写逻辑，监听 Syncthing 目录变化。

### 阶段 2：Electron 主进程与交互桥接
1.  **`main.js`**: Electron 入口，管理窗口生命周期。
2.  **`preload.js`**: 安全桥接层（Context Bridge），将文件和加密 API 暴露给前端，保证安全性。

### 阶段 3：前端 UI 开发 (HTML/CSS)
1.  **设置页 (Setup View)**：
    *   首次启动引导：输入用户名、选择同步文件夹、设置/导入团队密钥。
2.  **主面板 (Team Dashboard)**：
    *   左侧/顶部：团队成员头像/列表。
    *   主视图：展示全员今日状态概览。
3.  **个人每日看板 (My Daily Popup)**：
    *   **双栏布局**：左侧「今日目标」，右侧「实际完成」。
    *   **交互逻辑**：
        *   一键「同步目标」：将左侧内容复制到右侧。
        *   状态标记：右侧新增项自动添加 `.new-item` (绿底)，未完成项添加 `.failed-item` (红底)。

### 阶段 4：打包配置
1.  **`electron-builder` 配置**: 设置打包脚本，生成 Windows `.exe` 可执行文件（免安装版或安装版）。
